vupdaters

Introduction

vupdaters is a set of open-source tools for controlling the VU-1 USB dials from Streacom, written in Rust. Currently, vupdaters consists of two binary applications:

  • dialctl, a command-line tool for querying information about the dials connected to the system and manually setting their values,

  • vupdated, a daemon that runs in the background and continually updates the dials to display system information, based on a config file.

Both of these applications depend on VU-Server, which provides an HTTP API for configuring the dials. For more information on how to install and configure VU-Server, see the official documentation on vudials.com.

To install vupdaters, see here.

Using dialctl

dialctl is a command-line tool for interacting with the VU-Server HTTP API. You can use dialctl to list all VU-1 dials connected to your system, to show detailed status information about the dials, and to set a dial's value, backlight color, background image, and easing configuration.

All dialctl commands require a VU-Server API key. The API key is either read from the $VU_SERVER_API_KEY, or from the --key CLI option. The address of the VU-Server instance can be configured by the --server CLI option.

For more detailed usage information, use dialctl help.

command-line usage

A command-line tool for controlling Streacom VU-1 dials.

Use dialctl list to list all dials connected to the system, dialctl status to get detailed status information about a dial, or dialctl set to set a dial's value, backlight configuration, and background image.

dialctl [OPTIONS] --key <KEY> [COMMAND]

subcommands

  • list: List all dials
  • status: Get detailed status information about a dial
  • set: Set a dial's value, image file, backlight, or easing config
  • reload: Forcibly reload a dial's hardware info
  • help: Print this message or the help of the given subcommand(s)

options

  • -k, --key <KEY>: The server API key
    • env: VU_DIALS_API_KEY
  • -s, --server <SERVER>: The hostname of the VU-Server instance to connect to
    • default: http://localhost:5340
    • env: VU_DIALS_SERVER_ADDR
  • --trace <FILTER>: A list of log-level filters for tracing-subscriber
    • default: info
    • env: RUST_LOG

Setting dial state

The dialctl set command can be used to set a dial's value, background image, backlight color, and easing configuration.

command-line usage

Set a dial's value, image file, backlight, or easing config.

At least one of --value, --image, --red, --green, or --blue must be provided.

set [OPTIONS] <--dial <UID>|--index <INDEX>|--name <NAME>> <--value <VALUE>|--image <IMAGE>|--red <RED>|--green <GREEN>|--blue <BLUE>>

options

  • -d, --dial <UID>: Select a dial by its UID
  • -i, --index <INDEX>: Select a dial by its numeric index
  • -n, --name <NAME>: Select a dial by its user-assigned name
  • -v, --value <VALUE>: Set the dial's needle to the provided value. Values must be between 0 and 100.
  • --image <IMAGE>: Set the dial's background image to the provided image file
  • -r, --red <RED>: Set the red value of the dial's backlight to the provided value. Values must be between 0 and 100.
  • -g, --green <GREEN>: Set the green value of the dial's backlight to the provided value. Values must be between 0 and 100.
  • -b, --blue <BLUE>: Set the blue value of the dial's backlight to the provided value. Values must be between 0 and 100.
  • -s, --server <SERVER>: The hostname of the VU-Server instance to connect to
    • default: http://localhost:5340
    • env: VU_DIALS_SERVER_ADDR
  • --trace <FILTER>: A list of log-level filters for tracing-subscriber
    • default: info
    • env: RUST_LOG

Using vupdated

vupdated is a daemon process that runs in the background and continually updates the VU-1 dials to display system statistics, such as CPU load and temperature, memory usage, swap space usage, et cetera.

On Linux systems, vupdated can also provide USB hotplug support for VU-Server.

command-line usage

Daemon for updating VU-1 dials

vupdated [OPTIONS] --key <KEY> [COMMAND]

subcommands

  • gen-config: Generate a new config file with the given metrics
  • help: Print this message or the help of the given subcommand(s)

options

  • -c, --config <CONFIG_PATH>: Path to the config file
    • default: /home/runner/.config/vupdate/config.toml
  • -k, --key <KEY>: The server API key
    • env: VU_DIALS_API_KEY
  • -s, --server <SERVER>: The hostname of the VU-Server instance to connect to
    • default: http://localhost:5340
    • env: VU_DIALS_SERVER_ADDR
  • --trace <FILTER>: A list of log-level filters for tracing-subscriber
    • default: info
    • env: RUST_LOG
  • --hotplug-service <HOTPLUG_SERVICE>: The systemd unit name for the VU-Server service. When a hotplug event for a USB-serial device occurs, vupdated will attempt to restart this systemed service.
    • default: VU-Server.service

Configuration

Generating config files

The vupdated gen-config subcommand can be used to generate a default configuration file. Optionally, a list of desired metrics to display can be provided; when this is present, the generated config file will include those metrics.

command-line usage

Generate a new config file with the given metrics

gen-config [OPTIONS] [METRICS]...

arguments

  • [METRICS]...: The list of requested metrics to include in the config file. If more metrics are requested than the number of dials connected to the system, only the first N metrics will be included in the config, where N is the number of dials discovered.
    • default: cpu-load mem cpu-temp swap
    • possible values:
      • cpu-load : Display CPU load as a percentage
      • mem : Display memory usage, as a percentage of total memory
      • disk-usage : Display disk usage as a percentage of total disk space
      • cpu-temp : Display CPU temperature
      • swap : Display swap usage, as a percentage of total swap space
      • battery : Display the current remaining battery percentage

options

  • -c, --config <CONFIG_PATH>: Path to the config file
    • default: /home/runner/.config/vupdate/config.toml
  • -s, --server <SERVER>: The hostname of the VU-Server instance to connect to
    • default: http://localhost:5340
    • env: VU_DIALS_SERVER_ADDR
  • --trace <FILTER>: A list of log-level filters for tracing-subscriber
    • default: info
    • env: RUST_LOG
